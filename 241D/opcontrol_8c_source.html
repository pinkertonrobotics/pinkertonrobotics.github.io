37 /*
38  * Runs the user operator control code. This function will be started in its own task with the
39  * default priority and stack size whenever the robot is enabled via the Field Management System
40  * or the VEX Competition Switch in the operator control mode. If the robot is disabled or
41  * communications is lost, the operator control task will be stopped by the kernel. Re-enabling
42  * the robot will restart the task, not resume it from where it left off.
43  *
44  * If no VEX Competition Switch or Field Management system is plugged in, the VEX Cortex will
45  * run the operator control task. Be warned that this will also occur if the VEX Cortex is
46  * tethered directly to a computer via the USB A to A cable without any VEX Joystick attached.
47  *
48  * Code running in this task can take almost any action, as the VEX Joystick is available and
49  * the scheduler is operational. However, proper use of delay() or taskDelayUntil() is highly
50  * recommended to give other tasks (including system tasks such as updating LCDs) time to run.
51  *
52  * This task should never exit; it should end with some kind of infinite loop, even if empty.
53  */
